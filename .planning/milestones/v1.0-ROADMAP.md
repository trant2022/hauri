# Milestone v1.0: MVP

**Status:** SHIPPED 2026-03-06
**Phases:** 1-5
**Total Plans:** 15

## Overview

Unlockt is built in five phases that follow the critical path of value delivery: first establish the foundation and auth system, then enable creators to upload files and create payment links, then wire up the purchase-to-download money flow, then connect creators to payouts via Stripe Connect, and finally polish with dashboard analytics, creator profiles, and mobile responsiveness. Each phase delivers a complete, verifiable capability -- a creator can progressively do more real things after each phase ships.

## Phases

**Depends on**: Nothing (first phase)
**Requirements**: AUTH-01, AUTH-02, AUTH-03, AUTH-04, UX-01, UX-04, UX-05, PAGE-01
**Plans**: 3 plans

Plans:

## Milestone Summary

**Key Decisions:**

- Next.js 16.1.6 (latest, not 15) -- middleware deprecated but functional
- Zod v4 -- API compatible, uses `error` string for enum validation
- Dark-mode-first with OKLCH color system
- Stripe lazy Proxy pattern for build-time safety
- Separate charges and transfers (not direct charges) for Stripe Connect
- HMAC tokens with base64url + timingSafeEqual for download security
- Dual webhook secret verification for Connect events
- Transfer error isolation -- never blocks checkout or email
- Reserved username blocklist for /[username] route safety

**Issues Resolved:**
- Vercel serverless timeout: bypassed with direct-to-Supabase TUS uploads
- Stripe SDK build crash: lazy Proxy pattern defers initialization
- Webhook retry storms: always return 200 on handler errors
- Race conditions: PostgreSQL 23505 unique constraint for idempotent transactions
- Connect fund timing: source_transaction on transfers

**Technical Debt Incurred:**
- Next.js 16 middleware deprecation (functional, needs migration to "proxy" convention)
- Supabase email templates need manual dashboard configuration
- Using onboarding@resend.dev sender (production needs custom domain)
