  <name>Task 1: Webhook route with signature verification and event dispatch</name>
  <files>
    src/app/api/webhooks/stripe/route.ts
  </files>
  <action>
    Create `src/app/api/webhooks/stripe/route.ts`:

    1. Import: headers from "next/headers", NextResponse from "next/server", Stripe from "stripe", stripe from "@/lib/stripe/client"
    2. Import: handleCheckoutCompleted, handleDisputeCreated from "@/lib/stripe/webhooks"
    3. Read STRIPE_WEBHOOK_SECRET from process.env

    4. Export async function POST(req: Request):
       - Get raw body with `await req.text()` -- CRITICAL: never use req.json(), it breaks signature verification
       - Get signature from headers: `(await headers()).get("stripe-signature")`
       - If no signature, return 400
       - Wrap constructEvent in try/catch:
         - `const event = stripe.webhooks.constructEvent(body, signature, webhookSecret)`
         - On error: log "Webhook signature verification failed", return 400

    5. Dispatch by event.type using switch:
       - "checkout.session.completed": await handleCheckoutCompleted(event.data.object as Stripe.Checkout.Session)
       - "charge.dispute.created": await handleDisputeCreated(event.data.object as Stripe.Dispute)
       - default: log unhandled event type (not an error)

    6. Return NextResponse.json({ received: true }) with status 200

    7. Wrap the dispatch in try/catch so handler errors don't prevent the 200 response -- log the error but still return 200 to prevent Stripe retries for application errors (only return non-200 for signature failures).

    IMPORTANT: Do NOT use any body parser middleware. The raw text body is essential for signature verification.
  </action>
  <verify>
    - `npx tsc --noEmit` passes
    - File uses `req.text()` not `req.json()`
    - File imports handleCheckoutCompleted and handleDisputeCreated
  </verify>
  <done>Webhook route verifies Stripe signature using raw body text, dispatches checkout.session.completed and charge.dispute.created events to handler functions, returns 200 for all verified events.</done>

<task type="auto">
  <name>Task 2: Event handler functions (checkout completed + dispute created)</name>
  <files>
    src/lib/stripe/webhooks.ts
  </files>
  <action>
    Create `src/lib/stripe/webhooks.ts`:

    1. Import: Stripe from "stripe", supabaseAdmin from "@/lib/supabase/admin"

    2. Export async function handleCheckoutCompleted(session: Stripe.Checkout.Session):
       - Extract metadata: link_id, base_price, platform_fee, creator_amount from session.metadata!
       - Extract buyer email: session.customer_details?.email ?? session.customer_email ?? ""
       - Extract payment_intent: typeof session.payment_intent === "string" ? session.payment_intent : session.payment_intent?.id ?? null

       - Idempotent insert into transactions:
         ```
         const { error } = await supabaseAdmin.from("transactions").insert({
           link_id,
           buyer_email: buyerEmail,
           amount_paid: session.amount_total!,
           platform_fee: Number(platform_fee),
           creator_amount: Number(creator_amount),
           currency: session.currency!.toUpperCase(),
           stripe_session_id: session.id,
           stripe_payment_intent_id: paymentIntentId,
           status: "completed",
         })
         ```

       - Check error: if error.code === "23505" (unique_violation on stripe_session_id), log "Duplicate webhook, transaction already recorded" and return early -- this is expected and OK
       - If other error, throw it

       - After successful insert, increment unlock_count:
         `await supabaseAdmin.rpc("increment_unlock_count", { link_id_param: link_id })`

       - Log: "Transaction recorded for session: {session.id}"

       NOTE: Email sending will be wired in Plan 04. For now, add a comment: // TODO: Send purchase receipt email (Plan 03-04)

    3. Export async function handleDisputeCreated(dispute: Stripe.Dispute):
       - Extract payment_intent_id: typeof dispute.payment_intent === "string" ? dispute.payment_intent : dispute.payment_intent?.id
       - If no paymentIntentId, log warning and return

       - Update transaction to disputed status:
         ```
         const { data: transaction, error } = await supabaseAdmin
           .from("transactions")
           .update({ status: "disputed" })
           .eq("stripe_payment_intent_id", paymentIntentId)
           .select("id, link_id")
           .single()
         ```

       - If no transaction found, log "Dispute for unknown transaction" with paymentIntentId and return
       - If error, throw it

       - Decrement unlock_count:
         `await supabaseAdmin.rpc("decrement_unlock_count", { link_id_param: transaction.link_id })`

       - Log: "Dispute recorded, download access revoked for transaction: {transaction.id}"
  </action>

    - Both functions use supabaseAdmin (service role) not user client
  </verify>
  <done>handleCheckoutCompleted idempotently records transactions with all fee metadata and payment_intent_id, increments unlock_count. handleDisputeCreated revokes access by marking transaction disputed and decrementing unlock_count. Both use atomic RPC functions for count management.</done>
