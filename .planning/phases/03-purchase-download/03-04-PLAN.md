  <files>

  <action>
    1. Install resend: `npm install resend`

    2. Create `src/lib/email/templates/purchase-receipt.tsx`:
       - This is a React Email template rendered server-side by Resend. It does NOT need "use client".
       - Since we are NOT installing @react-email/* packages (too heavy for this use case), build the template as a simple React component that Resend renders to HTML.
       - Resend's `react` option accepts any React element and renders it server-side.

       - Export function PurchaseReceiptEmail(props):
         Props: { linkTitle: string, amountFormatted: string, downloadUrl: string }

       - Template content (using inline styles for email compatibility -- this is the ONE exception to the "no inline styles" rule, emails require them):
         - Container div with max-width 600px, centered, font-family sans-serif
         - Header: "Unlockt" brand text
         - Heading: "Your purchase receipt"
         - Body paragraphs:
           - "Thank you for your purchase!"
           - "Item: {linkTitle}"
           - "Amount paid: {amountFormatted}"
         - Download button/link: styled as a button linking to downloadUrl
           - Text: "Download your file"
           - Note below: "This link expires in 48 hours."
         - Footer: "If you have any issues, reply to this email." and "Powered by Unlockt"

       - Keep the template simple and clean. Dark background (#09090b) with light text (#fafafa) to match the brand, OR use light email template (white bg, dark text) which is more standard for transactional emails. Go with light template for email (better email client compatibility).

    3. Create `src/lib/email/send-receipt.ts`:
       - Import: Resend from "resend"
       - Import: PurchaseReceiptEmail from "./templates/purchase-receipt"
       - Import: createDownloadToken from "@/lib/download-token"
       - Import: formatPrice from "@/lib/fees"

       - Initialize: const resend = new Resend(process.env.RESEND_API_KEY)

       - Define interface SendReceiptParams:
         - buyerEmail: string
         - transactionId: string
         - linkId: string
         - linkTitle: string
         - amountPaid: number (in cents)
         - currency: string

       - Export async function sendPurchaseReceipt(params: SendReceiptParams):
         - Create download token: createDownloadToken(params.transactionId, params.linkId)
         - Build download URL: `${process.env.NEXT_PUBLIC_APP_URL}/api/download/${token}`
         - Format amount: formatPrice(params.amountPaid, params.currency)

         - Send email via Resend:
           ```
           const { error } = await resend.emails.send({
             from: "Unlockt <onboarding@resend.dev>",
             to: params.buyerEmail,
             subject: `Your purchase: ${params.linkTitle}`,
             react: PurchaseReceiptEmail({
               linkTitle: params.linkTitle,
               amountFormatted,
               downloadUrl,
             }),
           })
           ```
         - If error, throw it
         - NOTE: Use "onboarding@resend.dev" as the from address for development. This is Resend's shared dev sender that works without domain verification. Production will need a custom domain.
  </action>

    - send-receipt.ts imports formatPrice (proper price formatting)
  </verify>
