  <name>Task 1: HMAC download token module</name>
  <files>
    src/lib/download-token.ts
  </files>
  <action>
    Create `src/lib/download-token.ts` using Node.js built-in crypto (no external dependencies):

    1. Import: createHmac, timingSafeEqual from "crypto"
    2. Read DOWNLOAD_TOKEN_SECRET from process.env
    3. Define TOKEN_TTL_MS = 48 * 60 * 60 * 1000 (48 hours)

    4. Define interface TokenPayload:
       - transactionId: string
       - linkId: string
       - exp: number (expiry timestamp in ms)

    5. Export function createDownloadToken(transactionId: string, linkId: string): string
       - Build payload: { transactionId, linkId, exp: Date.now() + TOKEN_TTL_MS }
       - Encode payload: Buffer.from(JSON.stringify(payload)).toString("base64url")
       - Create signature: createHmac("sha256", SECRET).update(data).digest("base64url")
       - Return `${data}.${signature}`

    6. Export function verifyDownloadToken(token: string): TokenPayload | null
       - Split token on "." into [data, signature]
       - If either missing, return null
       - Compute expected signature: createHmac("sha256", SECRET).update(data).digest("base64url")
       - Compare using timingSafeEqual (constant-time to prevent timing attacks):
         - Convert both to Buffers with Buffer.from(x, "base64url")
         - If lengths differ, return null
         - If !timingSafeEqual(sigBuffer, expectedBuffer), return null
       - Parse payload: JSON.parse(Buffer.from(data, "base64url").toString()) as TokenPayload
       - Check expiration: if Date.now() > payload.exp, return null
       - Return payload

    IMPORTANT: Use base64url encoding (URL-safe, no padding) so tokens work in URLs without encoding issues.
    IMPORTANT: Use timingSafeEqual to prevent timing attacks -- do NOT use === for signature comparison.
  </action>
  <verify>
    - `npx tsc --noEmit` passes
    - createDownloadToken and verifyDownloadToken are exported
    - File uses timingSafeEqual, not === for signature comparison
    - File uses base64url encoding
  </verify>
  <done>Download token module creates HMAC-signed tokens with 48h expiry and verifies them with constant-time comparison. Tokens are URL-safe (base64url) and stateless (no database lookup needed for verification).</done>

  <name>Task 2: Success page + re-download API route</name>

  <action>
    1. Create `src/app/l/success/page.tsx` as a SERVER component (no "use client"):
       - Import: stripe from "@/lib/stripe/client", supabaseAdmin from "@/lib/supabase/admin", notFound from "next/navigation"
       - Import: formatPrice from "@/lib/fees"
       - Import UI components: Card, CardContent, Button from shadcn, CheckCircle2 and Download from lucide-react

       - Export default async function SuccessPage with searchParams: Promise<{ session_id?: string }>
       - Await searchParams, extract session_id
       - If no session_id, call notFound()

       - Retrieve Checkout session from Stripe API:
         `const session = await stripe.checkout.sessions.retrieve(session_id)`
         This handles the race condition -- we verify payment directly with Stripe, not the database.

       - If session.payment_status !== "paid":
         Render a "Payment Pending" state with a message like "Your payment is being processed. You will receive an email with your download link shortly." Style with the same card layout as the purchase page.

       - If payment_status === "paid":
         - Extract link_id from session.metadata
         - Fetch link with file data: supabaseAdmin.from("links").select("title, files(storage_path, name, size_bytes)").eq("id", link_id).single()
         - If no link/file found, show error state
         - Generate signed URL: supabaseAdmin.storage.from("files").createSignedUrl(storagePath, 60, { download: fileName })
           The { download: fileName } option ensures the browser downloads with the original filename.
         - Render success UI:
           - Green checkmark icon
           - "Payment Successful!" heading
           - "Thank you for your purchase" subtext
           - Card showing: file name, amount paid (formatted with formatPrice using session.amount_total and session.currency), currency
           - Download button: an <a> tag with href={signedUrl} and download attribute, styled as a shadcn Button with Download icon
           - Note: "Your download link expires in 60 seconds. Check your email for a link that works for 48 hours."
           - "Back to Unlockt" link to /

       - Style with dark-mode-first Tailwind classes matching the existing link page aesthetic.
       - Use min-h-screen centered layout like the link page.

    2. Create `src/app/api/download/[token]/route.ts`:
       - Import: NextResponse from "next/server", verifyDownloadToken from "@/lib/download-token", supabaseAdmin from "@/lib/supabase/admin"

       - Export async function GET(_req: Request, { params }: { params: Promise<{ token: string }> }):
         - Await params, extract token
         - Call verifyDownloadToken(token)
         - If null (invalid or expired): return JSON { error: "Invalid or expired download link" } with status 403

         - Verify transaction exists and is not disputed:
           ```
           const { data: transaction } = await supabaseAdmin
             .from("transactions")
             .select("link_id, status")
             .eq("id", payload.transactionId)
             .single()
           ```
         - If no transaction or status === "disputed": return JSON { error: "Download access revoked" } with status 403

         - Verify link_id matches (prevents token tampering):
           If transaction.link_id !== payload.linkId, return 403

         - Get file path:
           ```
           const { data: link } = await supabaseAdmin
             .from("links")
             .select("files(storage_path, name)")
             .eq("id", payload.linkId)
             .single()
           ```
         - If no link or no files, return 404

         - Generate fresh signed URL (60s):
           `supabaseAdmin.storage.from("files").createSignedUrl(storagePath, 60, { download: fileName })`

         - Return NextResponse.redirect(signedUrl) to trigger the download

    IMPORTANT: The success page is a server component -- it can await Stripe API calls directly.
    IMPORTANT: The download route generates a FRESH signed URL each time -- the email does NOT contain a signed URL, it contains the HMAC token.
    IMPORTANT: Use { download: fileName } in createSignedUrl to preserve original filenames.
  </action>
