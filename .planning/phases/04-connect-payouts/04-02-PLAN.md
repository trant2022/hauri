<objective>
Build the creator-facing Connect onboarding UI and earnings dashboard: API routes for initiating/resuming Stripe Connect onboarding, a settings page showing Connect account status with state machine display, and an earnings page showing total earned/transferred/pending amounts with a re-engagement prompt for incomplete onboarding.

Purpose: This gives creators the ability to actually onboard to Stripe Connect and see their earnings. Without this, the backend infrastructure from Plan 01 has no user-facing interface. The settings page is the entry point to the Connect flow, and the earnings page shows creators their money.

Output: Two API routes (onboard + refresh), settings page with Connect status component, earnings page with transfer totals, updated sidebar navigation, and a new Supabase query for earnings aggregation.
</objective>

  <action>
    **`src/app/api/connect/onboard/route.ts` (NEW):**

    POST endpoint. Auth required (use `createClient` from `@/lib/supabase/server`, call `supabase.auth.getUser()`, return 401 if no user).

    Flow:
    1. Query user profile: `supabase.from("users").select("stripe_account_id, email").eq("id", user.id).single()`
    2. If user has no `stripe_account_id`, create Express account using `createExpressAccount(profile.email)` from `@/lib/stripe/connect`. Save the account ID: `supabase.from("users").update({ stripe_account_id: accountId }).eq("id", user.id)`
    3. Generate fresh Account Link using `createAccountLink(accountId, refreshUrl, returnUrl)`:
       - `refreshUrl`: `${process.env.NEXT_PUBLIC_APP_URL}/api/connect/refresh`
       - `returnUrl`: `${process.env.NEXT_PUBLIC_APP_URL}/dashboard/settings?connect=return`
    4. Return `{ url: accountLink }` as JSON response
    5. Wrap in try/catch, return `{ error: string }` with 500 on failure

    **`src/app/api/connect/refresh/route.ts` (NEW):**

    GET endpoint (this is the `refresh_url` Stripe redirects to when an Account Link expires).

    Flow:
    1. Auth check -- if no user, redirect to `/login`
    2. Query user's `stripe_account_id`
    3. If no stripe_account_id, redirect to `/dashboard/settings`
    4. Generate fresh Account Link using `createAccountLink` with same URLs as the onboard route
    5. Redirect (302) to the new Account Link URL using `NextResponse.redirect(accountLinkUrl)`
    6. Wrap in try/catch, redirect to `/dashboard/settings?connect=error` on failure

    **`src/app/(dashboard)/dashboard/settings/page.tsx` (NEW):**

    Server component. This is the main settings page.

    1. Get authenticated user via `createClient` from `@/lib/supabase/server`
    2. Query user profile: select `email, username, stripe_account_id, charges_enabled, payouts_enabled, details_submitted, onboarding_complete`
    3. Derive Connect status from the state machine:
       - `NOT_STARTED`: No stripe_account_id
       - `ONBOARDING`: stripe_account_id exists, details_submitted is false
       - `PENDING`: details_submitted is true, charges_enabled is false
       - `ACTIVE`: charges_enabled is true AND payouts_enabled is true
       - `RESTRICTED`: Has stripe_account_id but charges_enabled is false AND details_submitted is true AND some time has passed (simplified: just use PENDING for now, RESTRICTED can be added when requirements.past_due data is available)
    4. Render page with heading "Settings", a section for account info (email, username -- read-only display), and the `<ConnectStatus>` client component passing `status` and a boolean `hasStripeAccount`.
    5. Check for `?connect=return` search param. If present, show a note that Connect status will update shortly (webhook may not have fired yet).

    **`src/app/(dashboard)/dashboard/settings/connect-status.tsx` (NEW):**

    `"use client"` component. Receives `status: 'NOT_STARTED' | 'ONBOARDING' | 'PENDING' | 'ACTIVE'` and `hasStripeAccount: boolean` as props.

    Display a Card (from shadcn/ui) with:
    - Title: "Stripe Connect"
    - Description based on status:
      - `NOT_STARTED`: "Connect your bank account to receive payouts from your sales."
      - `ONBOARDING`: "You started onboarding but haven't finished. Resume to start receiving payouts."
      - `PENDING`: "Your account is under review by Stripe. This usually takes 1-2 business days."
      - `ACTIVE`: "Your account is active. Payouts are automatically deposited to your bank account."
    - Status badge (use a colored dot + text):
      - `NOT_STARTED`: gray dot, "Not connected"
      - `ONBOARDING`: yellow dot, "Incomplete"
      - `PENDING`: yellow dot, "Under review"
      - `ACTIVE`: green dot, "Active"
    - Action button:
      - `NOT_STARTED`: "Connect with Stripe" primary button
      - `ONBOARDING`: "Resume onboarding" primary button
      - `PENDING`: No button (just waiting)
      - `ACTIVE`: No button (show "Payouts are automatic" note instead)
    - Button click handler: POST to `/api/connect/onboard`, get `{ url }` response, redirect via `window.location.href = url` (not Next.js router -- we're leaving the app to go to Stripe). Show loading state with Loader2 spinner while waiting. Show toast on error.
    - For ACTIVE status, add a small note: "Stripe automatically deposits your earnings to your bank account daily."
