  <action>
    **Step 1: Create link validation schema** (`src/lib/validations/link.ts`)

    Use the schema from the research doc's "Link Validation Schema" section as the foundation. Key details:
    - `createLinkSchema`: fileId (uuid), title (1-100 chars), description (optional, max 500 chars), priceAmount (positive integer, in cents -- the form will convert from display format), priceCurrency (enum of SUPPORTED_CURRENCIES from fees.ts), maxUnlocks (optional positive integer), previewUrl (optional valid URL).
    - `updateLinkSchema`: Partial of createLinkSchema plus `isActive` (optional boolean). Do NOT allow changing fileId on update.
    - Export types: `CreateLinkInput`, `UpdateLinkInput`.
    - Also export a `priceDisplaySchema` for the form: validates user input as a string that represents a positive number with max 2 decimal places (e.g., "10.50"), min value "0.50" (50 cents minimum). The form component will use this for client-side validation and convert to cents before submission.

    **Step 2: Create links API route** (`src/app/api/links/route.ts`)

    - **GET**: Auth check, call `getUserLinks(supabase, user.id)`, return JSON array. The query includes the joined file data (name, mime_type, size_bytes).
    - **POST**: Auth check, validate body with `createLinkSchema`, verify the file_id belongs to the user (SELECT from files where id = body.fileId AND user_id = user.id), generate slug with `nanoid(12)` (12 chars for URL-friendly short slugs), call `createLink()` with all fields plus generated slug and user_id. Catch unique constraint violation on slug and retry with new nanoid (max 3 attempts). Return 201 with created link.

    Import nanoid: `import { nanoid } from 'nanoid'`

    **Step 3: Create link detail API route** (`src/app/api/links/[linkId]/route.ts`)

    - **GET**: Auth check, fetch link by ID with ownership check (user_id matches). Return the link data including joined file info. Return 404 if not found or not owned by user.
    - **PATCH**: Auth check, validate body with `updateLinkSchema`, call `updateLink(supabase, linkId, user.id, validatedData)`. Return updated link. The user_id check in the query ensures ownership.
    - **DELETE**: This actually deactivates (not hard-deletes). Auth check, call `updateLink(supabase, linkId, user.id, { is_active: false })`. Return 200 with `{ success: true }`.

    Note: In Next.js 16, route params are accessed via `params` which is a `Promise`. Must `await` params: `const { linkId } = await params`.

    All routes follow the error handling pattern: try/catch wrapper, return `{ error: string }` on failure with appropriate status codes (400 for validation, 401 for auth, 404 for not found, 500 for internal).
  </action>

    **Step 1: Create link form component** (`src/components/link-form.tsx`)

    "use client" component. Dual-purpose: create and edit modes (prop `mode: 'create' | 'edit'`, optional `initialData` prop for edit mode, optional `linkId` for edit).

    Form fields (using React Hook Form + zodResolver):
    - **File selector** (only in create mode): shadcn Select dropdown listing user's files. Fetch from `/api/files` on mount. Show file name + size. Required.
    - **Title**: shadcn Input, required, placeholder "e.g., My Design Pack"
    - **Description**: shadcn Textarea, optional, placeholder "Describe what buyers will get..."
    - **Price**: Custom input with currency prefix. User types a decimal number (e.g., "10.50"). Validate with `priceDisplaySchema`. Show a live fee breakdown below the input using `calculateFees(Math.round(parseFloat(value) * 100))`:
      - "Buyer pays: {formatPrice(totalBuyerPays, currency)}"
      - "You receive: {formatPrice(creatorReceives, currency)}"
      Use a debounced calculation (or calculate on every change -- amounts are small computations).
    - **Currency**: shadcn Select with CHF, EUR, USD, GBP options. Default CHF.
    - **Preview image** (optional): Small upload zone (click to select, image only). Upload directly to Supabase `previews` bucket using standard (non-TUS) upload: `supabase.storage.from('previews').upload(path, file)`. Show preview thumbnail after upload. Store the public URL. **If preview upload fails, show a toast error and clear the preview field. Do not block form submission -- the creator can still submit the link without a preview.**
    - **Max unlocks** (optional): shadcn Input type number, placeholder "Unlimited"

    On submit (create mode): POST to `/api/links` with `{ fileId, title, description, priceAmount: Math.round(parseFloat(priceInput) * 100), priceCurrency, maxUnlocks: maxUnlocks || null, previewUrl }`. On success, show toast "Link created!" and redirect to `/dashboard/links` via `router.push()`.

    On submit (edit mode): PATCH to `/api/links/${linkId}` with changed fields. On success, toast "Link updated!" and redirect to `/dashboard/links`.

    Loading state on submit button. Disable form during submission.

    **Step 2: Create link list component** (`src/components/link-list.tsx`)

    "use client" component. Features:
    - Fetches links from `/api/links` on mount.
    - shadcn Table with columns: Title, File (name from joined data), Price (formatted with `formatPrice`), Status (Badge: green "Active" or gray "Inactive"), Unlocks (count/max or count/unlimited), Created.
    - Each row has actions via shadcn DropdownMenu: "Edit" (navigates to `/dashboard/links/{id}/edit`), "Copy Link" (copies `{window.location.origin}/l/{slug}` to clipboard, toast "Link copied!"), "Deactivate"/"Activate" toggle.
    - Deactivate: PATCH to `/api/links/{id}` with `{ isActive: false }`, refresh list. Activate: PATCH with `{ isActive: true }`.
    - Alternatively, show a Switch component inline in the Status column for quick toggle.
    - Empty state: "No payment links yet" with a "Create your first link" button linking to `/dashboard/links/new`.
    - Loading state: Skeleton rows.
