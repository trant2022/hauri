Add a new query function `getCreatorTransactionsWithLinks` to the Earnings Queries section of queries.ts. This query joins transactions with links to get per-link context:

```typescript
export async function getCreatorTransactionsWithLinks(
  supabase: SupabaseClient<Database>,
  userId: string
) {
  return supabase
    .from("transactions")
    .select(
      "id, creator_amount, currency, transfer_status, created_at, links!inner(id, title, slug, user_id)"
    )
    .eq("links.user_id", userId)
    .eq("status", "completed")
    .order("created_at", { ascending: false })
}
```

This differs from `getCreatorEarnings` by including link title and slug (needed for per-link breakdown table). The `!inner` join ensures only transactions for this creator's links are returned.

Do NOT modify `getCreatorEarnings` -- the earnings page uses it separately.
