    All three pages follow the same pattern: 'use client', React Hook Form with zodResolver, Supabase browser client, toast notifications, loading states via `isSubmitting`.

    **Signup page (`src/app/(auth)/signup/page.tsx`):**
    - Import `useForm` from react-hook-form, `zodResolver` from @hookform/resolvers/zod, `signupSchema`/`SignupInput` from @/lib/validations/auth, `createClient` from @/lib/supabase/client, `toast` from sonner, `useRouter` from next/navigation
    - Three fields: username, email, password — each using shadcn Input + Label components
    - Error display: below each field, show `errors.fieldName?.message` in text-sm text-destructive
    - Submit handler (`onSubmit`):
      1. Create Supabase client
      2. Check username uniqueness FIRST: `const { data: existing } = await supabase.from('users').select('username').eq('username', values.username).single()` — if `existing` is truthy, `toast.error('Username already taken')` and return
      3. Call `supabase.auth.signUp({ email: values.email, password: values.password, options: { data: { username: values.username }, emailRedirectTo: \`${window.location.origin}/auth/confirm\` } })`
      4. If error: `toast.error(error.message)`
      5. If success: `toast.success('Check your email to verify your account')`, then redirect to `/login` with a query param `?message=check-email` (or just show the toast and stay on page with a "check your email" message replacing the form)
    - Submit button: disabled when `isSubmitting`, shows "Creating account..." when loading, "Create account" normally. Use shadcn Button with `w-full`.
    - Links: "Already have an account? Log in" linking to /login

    **Login page (`src/app/(auth)/login/page.tsx`):**
    - Two fields: email, password
    - Submit handler:
      1. Create Supabase client
      2. Call `supabase.auth.signInWithPassword({ email: values.email, password: values.password })`
      3. If error: `toast.error(error.message)` — Supabase returns "Invalid login credentials" for wrong email/password
      4. If success: `toast.success('Welcome back!')`, `router.push('/dashboard')`, `router.refresh()` (refresh to update server component session state)
    - Submit button: "Sign in" / "Signing in..."
    - Links: "Don't have an account? Sign up" -> /signup, "Forgot password?" -> /forgot-password

    **Forgot password page (`src/app/(auth)/forgot-password/page.tsx`):**
    - One field: email
    - Submit handler:
      1. Create Supabase client
      2. Call `supabase.auth.resetPasswordForEmail(values.email, { redirectTo: \`${window.location.origin}/auth/confirm\` })`
      3. If error: `toast.error(error.message)`
      4. If success: `toast.success('Check your email for a reset link')`
      5. Show a "check your email" message replacing the form on success (use a useState boolean)
    - Submit button: "Send reset link" / "Sending..."
    - Link: "Back to login" -> /login

    **Auth confirm route handler (`src/app/auth/confirm/route.ts`):**
    - Use the exact pattern from research (Pattern 3)
    - GET handler that reads `token_hash`, `type`, and `next` from searchParams
    - If `token_hash` and `type` are present:
      1. Create Supabase server client (`await createClient()` from @/lib/supabase/server)
      2. Call `supabase.auth.verifyOtp({ type, token_hash })`
      3. If no error: redirect to the `next` param (default `/dashboard`)
      4. If error: redirect to an error page or `/login?error=verification-failed`
    - The `type` parameter distinguishes email verification (`type=email`) from password reset (`type=recovery`)
    - For `type=recovery`, the `next` param should be `/reset-password` (configured in the email template)
    - IMPORTANT: `verifyOtp` establishes an authenticated session via cookies — this is how the user is authenticated when they land on /reset-password

    **Auth callback route handler (`src/app/auth/callback/route.ts`):**
    - Placeholder for future OAuth flows
    - GET handler that reads `code` from searchParams
    - If `code` present: exchange for session using `supabase.auth.exchangeCodeForSession(code)`
    - Redirect to `/dashboard` on success, `/login?error=auth-failed` on error
    - This is a simple route handler — will be used if OAuth is added later

    **Reset password page (`src/app/(auth)/reset-password/page.tsx`):**
    - 'use client' component
    - Uses `resetPasswordSchema` with React Hook Form + zodResolver
    - Two fields: password, confirmPassword
    - Submit handler:
      1. Create Supabase browser client
      2. Call `supabase.auth.updateUser({ password: values.password })`
      3. This works because `verifyOtp` in the confirm route already established a session
      4. If error: `toast.error(error.message)`
      5. If success: `toast.success('Password updated successfully')`, `router.push('/login')`
    - Submit button: "Update password" / "Updating..."
    - Heading: "Set new password"
    - Description: "Enter your new password below"
    - Session check: On mount, verify user is authenticated via `supabase.auth.getUser()`. If NOT authenticated, redirect to `/login` (they shouldn't be on this page without a valid recovery session). Show a loading state while checking.
    - Styling: Same Card pattern as other auth forms
  </action>
